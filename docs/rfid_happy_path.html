Flujo de Acceso RFID (Happy Path)

Visualización del "Trigger" y Secuencia de Eventos cuando un empleado autorizado escanea su tarjeta en el sistema.

%%{init: {'theme': 'dark', 'sequence': {'showSequenceNumbers': true, 'messageAlign': 'center'} }}%% sequenceDiagram actor Usuario as "Usuario" participant Lector as "Lector RFID" participant Loop as "Task Background
(main.py)" participant Workflow as "process_swipe(...)
(domain)" participant DB as "SQLite DB" participant HW as "Hardware Controller
(Relé, LED, Buzzer)" Usuario->>Lector: 1. Pasa tarjeta RFID físicamente Lector-->>Loop: 2. Card detectada, retorna `card_id` Note over Loop: Ejecuta dentro de rfid_polling_task Loop->>Workflow: 3. Llama a lógica de negocio process_swipe(card_id...) Workflow->>DB: 4. Consulta si existe Usuario con ese `card_id` DB-->>Workflow: 5. Retorna entidad de Usuario (Autorizado = True) Workflow->>DB: 6. Guarda Registro de Acceso (Log exitoso) Note over Workflow,HW: Acciones de Hardware (Apertura) Workflow->>HW: 7. get_buzzer().beep() (Tono de éxito) Workflow->>HW: 8. get_green_led().turn_on() Workflow->>HW: 9. get_door_relay().turn_on() (Abre puerta) Note over HW: Espera configurada (ej. 5 segundos) Workflow->>HW: 10. get_door_relay().turn_off() (Cierra puerta) Workflow->>HW: 11. get_green_led().turn_off()
⚡

Trigger Asíncrono

El evento es iniciado de manera autónoma por el polling loop de `asyncio` sin bloquear peticiones a la web.

🔐

Validación Segura

El `domain workflow` abstrae y orquesta las lecturas y verificaciones contra la BD SQLite.

🚪

Acción de Hardware

Las interfaces sólidas de hardware encienden múltiples componentes sin depender los unos de los otros (SOLID).
